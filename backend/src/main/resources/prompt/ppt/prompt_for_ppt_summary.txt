<requirement2>
二、内容要求
1. 主标题：必须简洁明了，10-12字以内，直接回答"这一页到底想证明什么"
2. 副标题：补充背景或因果关系，让学生理解重要性
3. 核心要点：用并列结构，避免二级bullet，保证一眼读完
4. 关键数字/证据：把最具冲击力的1-2个关键词单独拎出来
5. 讲解提示：写给讲者自己看的2-3句话，确保台上不临场忘词
6. 衔接语：每页最后一行小字，提示"下一页我们将看到……"
</requirement2>

<requirement3>
三、生成规则
1. 基于讲义内容的整体结构生成总结
2. 重点关注核心概念、关键方法、重要结论
3. 对于知识点，突出其重要性和应用价值
4. 对于方法技巧，强调其适用场景和注意事项
5. 对于学习目标，明确掌握程度和后续方向
6. 语言表达要适合PPT展示，简洁有力
</requirement3>

<requirement4>
四、内容结构
1. 核心概念总结：突出重要定义和关键术语
2. 方法技巧总结：强调核心算法和解题思路
3. 应用场景总结：突出实际应用和工程价值
4. 学习目标总结：明确掌握程度和后续方向
5. 保持每页内容量适中，避免信息过载
</requirement4>

<requirement5>
五、PPT设计原则
1. 每页内容不超过4个要点，保证可读性
2. 关键概念和结论要突出显示
3. 总结内容要系统全面
4. 避免过于复杂的专业术语
5. 保持与前后页面的逻辑连贯性
6. 适合学生复习和巩固知识
</requirement5>

<requirement6>
六、总结页面特点
1. 系统性：全面概括所学内容
2. 重点性：突出核心概念和方法
3. 逻辑性：内容组织清晰有序
4. 实用性：便于学生复习巩固
5. 启发性：为后续学习指明方向
6. 完整性：涵盖所有重要知识点
</requirement6>
