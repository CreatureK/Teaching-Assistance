<requirement1>
一、PPT生成总体要求
1. 基于讲义内容的结构化组织，生成适合课堂教学的PPT
2. 每页PPT必须包含：主标题、副标题、核心要点、关键证据、讲解提示、衔接语
3. 确保PPT内容逻辑清晰，层次分明，便于学生理解和记忆
4. 遵循"一页一主题"原则，避免信息过载
5. 保持整体风格统一，便于教师讲解和学生理解
</requirement1>

<requirement2>
二、PPT页面结构要求
1. **主标题（Headline）**：1行，10-12字以内，回答"这一页到底想证明什么"
2. **副标题或过渡句（Sub-headline）**：1行，补充背景或因果关系，让学生知道"为什么这很重要"
3. **核心要点（Bullet Points）**：用并列结构；避免二级bullet，保证一眼读完
4. **关键数字/证据（Evidence）**：把最具冲击力的1-2个关键词单独拎出来，加括号或破折号跟在要点后面
5. **备注**：
   - 讲解提示（Speaker Note）：用PPT备注栏写给讲者自己看的2-3句话，确保台上不临场忘词
   - 衔接语（Transition）：每页最后一行小字，提示"下一页我们将看到……"，保持PPT连贯
</requirement2>

<requirement3>
三、PPT内容组织原则
1. **概览页面**：基于讲义overview部分，生成课程整体介绍
2. **基础内容页面**：基于讲义core_content_basic部分，生成基础概念和方法
3. **必备内容页面**：基于讲义core_content_essential部分，生成核心技能和技巧
4. **进阶内容页面**：基于讲义core_content_advanced部分，生成高级理论和应用
5. **例题习题页面**：基于讲义example_exercises部分，生成练习和巩固内容
6. **总结页面**：基于整体内容，生成知识总结和后续方向
</requirement3>

<requirement4>
四、PPT设计规范
1. **内容密度**：每页不超过4个要点，保证可读性
2. **语言表达**：简洁明了，避免复杂专业术语
3. **逻辑结构**：前后页面逻辑连贯，便于理解
4. **重点突出**：关键概念和数据要突出显示
5. **层次清晰**：内容层次分明，便于学生理解
6. **适合讲解**：为教师提供充分的讲解提示
</requirement4>

<requirement5>
五、生成流程指导
1. 首先分析讲义内容结构，确定PPT页面数量和类型
2. 为每种类型的页面选择合适的prompt模板
3. 确保每页内容符合模板要求，保持格式一致
4. 检查页面间的逻辑连贯性，优化衔接语
5. 验证整体内容的完整性和准确性
6. 确保生成的PPT适合课堂教学使用
</requirement5>

<requirement6>
六、质量保证要求
1. **内容准确性**：确保PPT内容与讲义内容一致
2. **逻辑清晰性**：页面间逻辑关系明确，便于理解
3. **表达简洁性**：语言表达简洁有力，避免冗余
4. **结构完整性**：涵盖讲义的所有重要内容
5. **实用性**：适合教师讲解和学生理解
6. **美观性**：内容布局合理，便于阅读
</requirement6>
