# 前端开发环境使用指南

## 项目概述

这是一个基于以下技术栈的前端项目：
- Vue 3
- TypeScript
- Vite
- Element Plus
- Vue Router

## 环境要求

- Node.js (建议使用最新LTS版本)
- Yarn 包管理工具

## 安装步骤

### 安装Node.js

1. 访问Node.js官网(https://nodejs.org/)下载最新的LTS版本
2. 按照安装向导完成安装
3. 安装完成后，打开命令行工具验证安装：
```bash
node -v
npm -v
```

### 安装Yarn

使用npm安装Yarn：
```bash
npm install -g yarn
```

安装完成后验证：
```bash
yarn -v
```

### 配置Yarn

设置Yarn镜像源以提高下载速度：
```bash
yarn config set registry https://registry.npmmirror.com
```

### 安装项目依赖

1. 进入项目目录：
```bash
cd frontend
```

2. 使用Yarn安装依赖：
```bash
yarn install
```

### 启动开发服务器

```bash
yarn dev
```
启动后可通过浏览器访问本地开发服务器（一般为 http://localhost:5173）

## 主要依赖包

- **Vue 3**: 渐进式JavaScript框架
- **Vue Router**: Vue的官方路由管理器
- **Element Plus**: 基于Vue 3的组件库
- **Axios**: 基于promise的HTTP客户端
- **@toast-ui/editor**: Markdown编辑器
- **katex & mathjax**: 数学公式渲染库

## 开发流程

1. 修改或添加组件：在`src/components/`目录下
2. 添加新页面：在`src/views/`目录下创建新的Vue组件，并在`router/index.ts`中添加路由配置
3. 添加新API调用：在`src/api/`目录下创建或修改相应的API服务

## 常见问题

### 安装依赖时出现网络问题

可以尝试配置Yarn镜像源：
```bash
yarn config set registry https://registry.npmmirror.com
```

# 后端开发环境使用指南

## 后端项目概述

这是一个基于以下技术栈的后端项目：
- Spring Boot 3.5.3
- Java 17
- Maven
- MyBatis-Plus
- MySQL
- JWT
- OpenAI API (通过阿里云DashScope兼容模式)

## 环境要求

- JDK 17或更高版本
- IntelliJ IDEA (推荐使用Ultimate版本，但Community版本也可以)
- MySQL 8.0或更高版本
- Maven 3.8+（IDEA内置或独立安装均可）

## 安装步骤

### 安装JDK

1. 访问Oracle官网(https://www.oracle.com/java/technologies/downloads/)或使用OpenJDK
2. 下载JDK 17或更高版本
3. 按照安装向导完成安装
4. 配置JAVA_HOME环境变量：
   - Windows:
     ```
     setx JAVA_HOME "C:\Program Files\Java\jdk-17"
     setx PATH "%PATH%;%JAVA_HOME%\bin"
     ```
   - macOS/Linux:
     ```
     export JAVA_HOME=/path/to/jdk
     export PATH=$PATH:$JAVA_HOME/bin
     ```
5. 验证安装：
   ```bash
   java -version
   ```

### 安装IntelliJ IDEA

1. 访问JetBrains官网(https://www.jetbrains.com/idea/download/)
2. 下载并安装IntelliJ IDEA（推荐Ultimate版本，学生可以申请免费许可证）
3. 启动IDEA并完成初始配置

### 配置IntelliJ IDEA

1. 配置JDK：
   - 打开IDEA，进入`File > Settings > Build, Execution, Deployment > Build Tools > Maven`
   - 设置`JDK for importer`为已安装的JDK 17
   - 进入`File > Project Structure > Project`，设置`Project SDK`为JDK 17

2. 配置Maven（如果使用IDEA内置的Maven，可跳过此步骤）：
   - 进入`File > Settings > Build, Execution, Deployment > Build Tools > Maven`
   - 设置`Maven home path`为Maven安装目录
   - 设置`User settings file`和`Local repository`为你的自定义路径（可选）

3. 配置Spring Boot支持：
   - 如果使用Ultimate版本，Spring支持已内置
   - 如果使用Community版本，需安装Spring插件：
     - 进入`File > Settings > Plugins`
     - 搜索并安装`Spring Boot Assistant`插件

### 安装MySQL

1. 访问MySQL官网(https://dev.mysql.com/downloads/mysql/)下载MySQL安装包
2. 按照安装向导完成安装
   - 记住设置的root用户密码
   - 选择"Developer Default"安装类型即可满足开发需求
3. 安装完成后验证服务是否启动：
   - Windows: 在服务管理中查看MySQL服务是否运行
   - macOS: 在系统偏好设置中查看MySQL服务是否运行
   - Linux: 运行`sudo systemctl status mysql`

### 配置数据库

1. 使用MySQL客户端或命令行登录MySQL：
   ```bash
   mysql -u root -p
   ```

2. 创建项目所需的数据库：
   ```sql
    -- 创建数据库
    CREATE DATABASE IF NOT EXISTS backend DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

    USE backend;

    -- 创建用户表
    CREATE TABLE IF NOT EXISTS user (
                                        id INT AUTO_INCREMENT PRIMARY KEY,
                                        username VARCHAR(50) NOT NULL COMMENT '用户名',
                                        password VARCHAR(255) NOT NULL COMMENT '密码（加密存储）',
                                        email VARCHAR(100) UNIQUE COMMENT '邮箱',
                                        avatar_url VARCHAR(255) COMMENT '头像URL',
                                        role VARCHAR(20) NOT NULL COMMENT '用户角色（teacher/admin）',
                                        status VARCHAR(20) NOT NULL DEFAULT 'active' COMMENT '账户状态（active/frozen）',
                                        is_deleted TINYINT(1) NOT NULL DEFAULT 0 COMMENT '删除标记 0-未删除,1-已删除',
                                        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
                                        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
                                        INDEX idx_email (email),
                                        INDEX idx_username (username)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

    -- 创建课程表
    CREATE TABLE IF NOT EXISTS course (
                                        id INT AUTO_INCREMENT PRIMARY KEY,
                                        teacher_id INT NOT NULL COMMENT '教师ID',
                                        name VARCHAR(100) NOT NULL COMMENT '课程名称',
                                        status VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT '审核状态',
                                        is_deleted TINYINT(1) NOT NULL DEFAULT 0 COMMENT '删除标记',
                                        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
                                        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
                                        FOREIGN KEY (teacher_id) REFERENCES user(id) ON DELETE CASCADE,
                                        INDEX idx_teacher (teacher_id)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='课程表';

    -- 创建课程目标表
    CREATE TABLE IF NOT EXISTS course_objective (
                                                    course_id INT PRIMARY KEY COMMENT '课程ID',
                                                    course_content TEXT COMMENT '课程内容',
                                                    teaching_target TEXT COMMENT '教学目标',
                                                    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
                                                    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
                                                    FOREIGN KEY (course_id) REFERENCES course(id) ON DELETE CASCADE
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='课程目标表';

    -- 创建教学大纲表
    CREATE TABLE IF NOT EXISTS syllabus (
                                            course_id INT PRIMARY KEY COMMENT '课程ID',
                                            content TEXT COMMENT '教学大纲内容',
                                            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
                                            updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
                                            FOREIGN KEY (course_id) REFERENCES course(id) ON DELETE CASCADE
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='教学大纲表';

    -- 创建讲义表
    CREATE TABLE IF NOT EXISTS material (
                                            course_id INT PRIMARY KEY COMMENT '课程ID',
                                            content TEXT COMMENT '讲义内容',
                                            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
                                            updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
                                            FOREIGN KEY (course_id) REFERENCES course(id) ON DELETE CASCADE
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='讲义表';

    -- 创建功能限制表
    CREATE TABLE IF NOT EXISTS restriction (
                                            id INT AUTO_INCREMENT PRIMARY KEY,
                                            user_id INT NOT NULL COMMENT '用户ID',
                                            function_name VARCHAR(50) NOT NULL COMMENT '功能名称',
                                            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
                                            FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE,
                                            UNIQUE KEY uk_user_function (user_id, function_name),
                                            INDEX idx_user (user_id)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='功能限制表';

    -- 插入初始管理员用户（密码需要加密后存储，这里使用明文示例）
    INSERT INTO user (username, password, email, role, status)
    VALUES ('admin', '123456', 'admin@example.com', 'admin', 'active');

    -- 插入示例教师用户
    INSERT INTO user (username, password, email, role, status)
    VALUES ('teacher', '123456', 'teacher@example.com', 'teacher', 'active');

   ```

### 导入项目

1. 启动IntelliJ IDEA
2. 选择`File > Open`，导航到项目的根目录（包含pom.xml的目录）
3. 等待IDEA自动导入Maven项目并下载依赖
   - 如果没有自动导入，右键点击`pom.xml`文件，选择`Add as Maven Project`

### 配置应用属性

1. 打开`src/main/resources/application.properties`
2. 修改数据库连接配置，确保与你的MySQL设置一致：
   ```properties
   spring.datasource.url=jdbc:mysql://localhost:3306/backend?useSSL=false
   spring.datasource.username=root
   spring.datasource.password=你的密码
   ```
### 运行项目

1. 在IDEA中，找到`com.java_web.backend.BackendApplication`类
2. 右键点击该类，选择`Run 'BackendApplication'`
3. 等待项目启动，控制台输出如下信息表示启动成功：
   ```
   Started BackendApplication in x.xxx seconds (process running for x.xxx)
   ```
4. 访问`http://localhost:8080`测试API（如果应用有Swagger UI，可访问`http://localhost:8080/swagger-ui/index.html`查看API文档）

## 常见问题

### 数据库连接问题

如果遇到数据库连接错误：
1. 确认MySQL服务已启动
2. 检查application.properties中的数据库连接信息
3. 确认数据库和用户存在并有正确权限

### 依赖下载失败

如果Maven依赖下载失败：
1. 检查网络连接
2. 配置Maven镜像源，在`~/.m2/settings.xml`中添加：
   ```xml
   <mirrors>
     <mirror>
       <id>aliyunmaven</id>
       <mirrorOf>central</mirrorOf>
       <name>阿里云公共仓库</name>
       <url>https://maven.aliyun.com/repository/public</url>
     </mirror>
   </mirrors>
   ```
3. IDEA中右键点击项目，选择`Maven > Reload Project`

### 端口占用问题

如果8080端口被占用：
1. 在`application.properties`中修改端口：
   ```properties
   server.port=8081
   ```
2. 或找到占用端口的进程并关闭：
   - Windows: `netstat -ano | findstr 8080`
   - Linux/macOS: `lsof -i :8080`

### Java版本不匹配

如果遇到Java版本错误：
1. 确认JDK版本为17或更高
2. 在IDEA中检查项目JDK设置
3. 检查pom.xml中的Java版本属性：
   ```xml
   <properties>
     <java.version>17</java.version>
   </properties>
   ```
